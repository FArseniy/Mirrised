# Автоматическое тестирование Mirrised

Тесты используют встроенный раннер Node.js (`node:test`) и `socket.io-client`.
Они запускают Express и Socket.IO на случайном локальном порту: настоящий экран,
браузер и TURN для этих сценариев не нужны.

```text
test/
├── helpers/socket.js   # подключение клиента, ACK и ожидание событий
└── rooms.test.js       # HTTP, комнаты, PIN, сигналинг и очистка
```

## Запуск

```bash
npm install
npm test
```

Тесты намеренно идут последовательно (`--test-concurrency=1`), потому что модель
комнат хранится в памяти одного процесса.

## Проверяемые сценарии

| Группа | Что подтверждает тест |
| --- | --- |
| HTTP | `GET /healthz` отвечает состоянием сервиса |
| Origin и proxy IP | принимается только разрешённый Origin; rate limit использует последний доверенный IP из заголовка |
| Валидация | пустой и некорректный `roomId` отклоняется |
| Участники | допускаются только ведущий и один зритель; третий получает `room-full` |
| Отключения | зритель может быть заменён; отключение ведущего закрывает комнату |
| Сигналинг | offer доходит только до второго участника той же комнаты; неверная роль отклоняется |
| PIN | после пяти неверных попыток ввод блокируется |
| Очистка | просроченные резервации и неактивные комнаты удаляются |

## Что проверяется вручную

- выбор экрана через `getDisplayMedia()` и отказ пользователя;
- автозапуск видео в целевых браузерах;
- адаптивность на телефоне и компьютере;
- передача между двумя разными сетями;
- маршрут TURN `relay` в `chrome://webrtc-internals`.

Для UI-автоматизации можно добавить Playwright позднее. В MVP не стоит подменять
системный диалог захвата экрана: ручная проверка на двух устройствах точнее и
безопаснее.
